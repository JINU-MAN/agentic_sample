아래처럼 구조 짜 봤는데 필요한거 있으면 설계 변경하고 추가해서 구현할 수 있겠어?


에이전틱 시스템의 메인 에이전트(플래너) 메인 에이전트에서는 
	start_agentic.py (하위 에이전트들 서버 실행, )
	session_memory.py (메인 에이전트 뿐 아니라 하위 에이전트와 주고받은 컨텍스트도 저장)
	agent.py(전체 에이전틱 시스템의 플래너 역할 수행)
	event_manager.py (a2a 바탕으로 이벤트 큐 만들어서 작업 수행, 하위 에이전트에게 작업 명령을 수행)
	system_logger.py (로깅 작업 수행)
	/log
		전체 로그jsonl
		세션로그.jsonl
		/세션로그폴더

하위 에이전트(paper, sns, web_search)
	/tool
		사용 가능한 도구들.py
	/well_known
		agent_card.json
	event_manager.py (이벤트 큐 만들어서 작업 수행, a2a 사용)
	/log
		세션 로그 폴더(각 세션마다)
	session_memory.py (session_store기능)
	user_entry_point.py(기본적으로 메인 에이전트와 소통하지만 단일 에이전트만 필요한 경우에)
	agent.py (각 에이전트의 플래너)
	
메인 에이전트를 실행하면 하위 에이전트들의 서버를 가동하고 브릿지 연결, 각 에이전트들의 에이전트 카드 확인 후 에이전트들이 수행 가능한 작업, 도구 컨텍스트 저장, 사용자의 입력이 들어오면 그 입력에 맞춰서 작업 명령 계획 이벤트 큐 생성, 각 이벤트에서 명령을 전달하고 명령을 전달받은 하위 에이전트는 해당 명령에 대해 각 에이전트의 도구, 능력에 따라 작업 계획 이벤트 큐 생성 후 작업 수행, 수행 결과와 추가 필요한 사항, 문제 상황들을 다시 메인 에이전트로 전달, 메시지를 받은 에이전트는 명령 이벤트 큐 수정이 필요하면 수정하거나 아니면 다음 이벤트 deque,
